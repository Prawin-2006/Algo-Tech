🧠 1. MAIN PROMPT (Paste this first in Replit AI)
Writing
Build a healthcare record management prototype using Python FastAPI for backend.
Features:
User registration (patient & doctor roles)
Upload patient medical data (store as JSON)
Encrypt data using AES before storing
Store encrypted data locally (simulate IPFS)
Generate SHA-256 hash of data (simulate blockchain storage)
QR code generation for patient ID
API to retrieve patient data using QR code
Role-based access:
Emergency: show only critical fields (allergies, blood group, diseases)
Full access: show complete record (only for authenticated doctors)
Chatbot endpoint:
Accept user query
Search patient data using keyword matching
Return relevant info (e.g., allergies, conditions)
Emergency override:
Allow doctor to access full data
Log reason and timestamp
Use FastAPI, pydantic models, and keep the code simple and modular.

Writing

Requirements:

Use keyword matching:
"allergy" → return allergy data
"blood" → return blood group
"disease" → return diseases
"medicine" → return prescriptions
If no match → return "No relevant data found"
